# 分阶段路线图

本文档用于约束后续开发顺序，避免在控制面尚未稳定时过早引入驱动、路由、NAT 或桥接。

## 阶段 1：安全 MVP 骨架

目标：让项目可运行、可配置、可诊断。

状态：已完成。

范围：

- WPF + MVVM 主结构。
- 首页、网卡、发现、配置、诊断、日志、教程页面。
- 本机网卡枚举。
- JSON 配置。
- 设备 ID。
- 日志服务。
- 平台服务接口占位。

不做：

- 不创建虚拟网卡。
- 不修改系统路由。
- 不做 NAT 或桥接。
- 不转发真实流量。

## 阶段 2：发现、会话与一键前置检查

目标：让两台电脑能互相发现，并建立安全的模拟连接状态。

状态：已完成 MVP。

范围：

- UDP 局域网设备发现。
- 调试端与网关端角色。
- 模拟握手。
- 单会话锁定。
- 对端离线后释放锁定。
- 端口占用检测。
- 自动选择可用端口。
- PID 和进程名识别。

不做：

- 阶段 2 本身不打开真实数据隧道。
- 阶段 2 本身不转发目标设备流量；阶段 3 已通过 TCP 代理完成应用层转发。

## 阶段 3：TCP 端口代理

目标：先实现第一条真实数据链路，让调试端通过本机 TCP 端口访问网关端后的目标设备端口。

状态：已完成 MVP。

已实现方式：

- 调试端监听 `127.0.0.1:LocalListenPort`。
- 网关端监听 `0.0.0.0:ProxyControlPort`。
- 调试代码连接这个本地端口。
- 调试端程序向网关端发送 `OpenConnection` 控制消息。
- 网关端连接 `TargetDeviceIp:TargetDevicePort`。
- 两端程序做双向字节流复制。

最小闭环：

- 一次只连接一个网关端。
- 一个会话内允许多个 TCP 客户端连接。
- 每个 TCP 客户端分配一个 `ConnectionId`。
- 连接关闭时释放该 `ConnectionId`。
- 日志和状态记录连接开始、结束、字节数和错误。

阶段 3 不做：

- 不创建虚拟网卡。
- 不模拟完整 IP 网段。
- 不处理 UDP 设备协议。
- 不做二层广播或组播。
- 阶段 3 本身不做 SharedKey 认证、加密、心跳或自动重连；这些已在阶段 4 完成。

## 阶段 4：认证、加密与稳定性

目标：让两端连接具备基础安全性和长期运行能力。

状态：已完成 MVP。

已实现范围：

- SharedKey 强制握手认证。
- 会话令牌。
- HMAC-SHA256 消息签名。
- AES-GCM 数据帧加密封装。
- nonce 和时间戳防重放。
- 心跳。
- 空闲超时断开。
- 连接失败重试。
- 关闭帧和断线清理。
- `AUDIT` 审计日志。

阶段 4 不做：

- 不做生产级证书体系。
- 不做自动密钥轮换。
- 不保证任意 TCP 流的透明断点续传；断线后由本地调试程序重新连接本机代理端口。

## 阶段 5：真实虚拟网段

目标：让调试端本机代码可以像访问本地网段一样访问远端目标设备网段。

状态：已完成 MVP。

已实现路线：

- 目标设备网段精确路由。
- 网关端 NAT。
- 操作预览。
- 应用脚本生成。
- 回滚脚本生成。
- WiFi、VPN 和正常上网共存检查。
- 禁止接管默认路由 `0.0.0.0/0`。
- 禁止静默修改系统 DNS 或 VPN DNS。
- 所有真实系统修改都只生成脚本，不由 WPF 程序静默执行。

当前未内置路线：

- Wintun/TUN + 三层路由。
- TAP/桥接，仅在必须支持二层广播时考虑。

已经满足的安全约束：

- 管理员权限显式确认。
- 操作预览。
- 回滚策略。
- WiFi、VPN 和正常上网共存检查。
- 禁止接管默认路由 `0.0.0.0/0`。
- 禁止静默修改系统 DNS 或 VPN DNS。
- 只允许添加目标设备网段的精确路由。

后续生产化增强：

- 脚本执行前后状态快照。
- 执行日志持久化。
- 更细粒度的回滚状态检测。
- 如必须支持驱动级透明虚拟 IP，再增加 Wintun/TUN 驱动安装和卸载策略。

## 阶段 6：生产发布

目标：从开发工具变成可交付工具。

状态：已完成发布工程化 MVP。

已实现范围：

- Portable zip 发布包。
- 自包含单文件发布包。
- GitHub Releases 更新检查。
- GitHub Release 更新包下载。
- 启动时后台检查更新。
- 崩溃日志。
- 持久化运行日志。
- 发布脚本。
- SHA256 校验文件。
- GitHub Actions 标签发布 workflow。
- SignTool 代码签名接入点。

当前边界：

- 代码签名证书需要发布者自行配置，不能提交到仓库。
- 当前不做静默自更新；更新包下载后由用户关闭程序并手动运行或替换。
- 当前发布形态是 portable zip 和自包含 exe，不内置 MSI/MSIX 图形安装器。
- 驱动级 Wintun/TUN 透明虚拟 IP 仍属于后续网络能力增强。
